### 📝 Sobre la Tarea: Creación de APIs en Node.js

Para esta asignación, la Profesora Naty ha diseñado un proyecto que pondrá a prueba la capacidad de los estudiantes para levantar un servidor desde cero y exponer datos de manera estructurada. El flujo conceptual que los estudiantes deben implementar se detalla a continuación:

![Flujo de desarrollo y APIs](https://images.unsplash.com/photo-1531403009284-440f080d1e12?auto=format&fit=crop&w=800&q=80)

#### 🎯 Objetivos principales de la tarea evaluada por la Prof. Naty:

1. **Inicialización del Proyecto:** Configurar correctamente el entorno de Node.js (usando `npm init`) y gestionar dependencias (como `express` y `nodemon`).
2. **Creación del Servidor:** Levantar un servidor local que escuche en un puerto específico.
3. **Definición de Endpoints:** Crear rutas claras y lógicas para diferentes recursos (ej. `/usuarios`, `/productos`).
4. **Manejo de Peticiones y Respuestas:** Recibir datos del cliente (a través de `req.body` o `req.params`) y devolver respuestas adecuadas en formato JSON.
5. **Manejo de Errores:** Implementar respuestas correctas cuando un recurso no se encuentra (Error 404) o hay un fallo en el servidor (Error 500).

> 💡 *"Una buena API es como un buen restaurante: el menú (los endpoints) debe ser claro, el mesero (el servidor) rápido, y la comida (los datos JSON) exactamente lo que el cliente pidió."* — **Prof. Naty**
